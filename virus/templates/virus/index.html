// Modelo SIR de Propagación de Malware

Para emular matemáticamente el brote de WannaCry se adapta el clásico modelo epidemiológico SIR (Susceptibles - Infectados - Recuperados) propuesto por Kermack y McKendrick. En este contexto informático, el contagio representa la replicación autónoma del gusano a través del puerto TCP 445 (SMB) y la recuperación representa la instalación del parche MS17-010 de Windows o la inactivación mediante el dominio Kill-Switch.

Ecuaciones Diferenciales Ordinarias (EDOs) Acopladas

El sistema dinámico de la epidemia digital está dado por:

$$\begin{align*} \frac{dS}{dt} &= -\frac{\beta S I}{N} \\ \frac{dI}{dt} &= \frac{\beta S I}{N} - \gamma I \\ \frac{dR}{dt} &= \gamma I \end{align*}$$

Donde las variables y los coeficientes dinámicos se definen como:

  • S(t): Computadoras vulnerables sin parche de seguridad que se encuentran expuestas.
  • I(t): Computadoras con infección activa ejecutando WannaCry y buscando propagarse.
  • R(t): Sistemas recuperados, inmunizados o desactivados por el freno de control.
  • β (Beta): Frecuencia de infección diaria (tasa de contacto efectivo por unidad de tiempo).
  • γ (Gamma): Tasa diaria de parcheo / recuperación.

Solución Numérica con Método Runge-Kutta 4 (RK4)

Debido a la no linealidad en el término de contagio \(\beta S I\), el sistema no admite una solución exacta en términos de funciones continuas estándar. Por tanto, el simulador del backend ejecuta un **algoritmo Runge-Kutta de 4º Orden (RK4)** para aproximar las trayectorias de forma precisa:

k₁ = f(t_n, y_n)
k₂ = f(t_n + h/2, y_n + h/2 * k₁)
k₃ = f(t_n + h/2, y_n + h/2 * k₂)
k₄ = f(t_n + h, y_n + h * k₃)
y_{n+1} = y_n + h/6 * (k₁ + 2k₂ + 2k₃ + k₄)

Número de Infección Básico (R₀)

El factor fundamental de propagación epidémica se calcula como:

$$R_0 = \frac{\beta}{\gamma}$$

Si \(R_0 > 1\), el malware se propagará exponencialmente en la red causando un brote masivo. Si \(R_0 \le 1\), el virus morirá gradualmente sin causar epidemia digital.
